# Intent-Spec Template

A Claude Code + OpenSpec template for spec-driven development. Changes start from
explicit intent, move through behaviour specs, design, and durable architecture
decisions, and end as implementation tasks that preserve the reasoning behind the
work.

Inspired by [intent-driven.dev](https://intent-driven.dev/blog/2026/05/10/spec-driven-development-openspec-opencode/)
and its [intent-driven-template](https://github.com/intent-driven-dev/intent-driven-template)
(built for OpenCode). This is an independent Claude Code port and generalization.

## Workflow

```text
proposal -> specs -> design -> adr -> tasks -> verify -> retrospective
```

- **proposal** — why the change matters (interrogated with the `grill-me` skill).
- **specs** — observable behaviour as OpenSpec deltas with Gherkin GIVEN/WHEN/THEN.
- **design** — implementation approach and trade-offs (C4 diagrams).
- **adr** — durable architecture decisions, written to `docs/adrs/` so they survive change-archiving.
- **tasks** — the work, derived from the above.
- **verify** — self-contained post-implementation check (`openspec validate` + task / spec / ADR coherence) written to `verify.md`.
- **retrospective** — evidence-first reflection written to `retrospective.md` (the archive-to-knowledge-base record). Both are plugin-independent.

**Optional — adversarial pass.** For a high-stakes artifact, run `/opsx:adversarial <artifact> <change>` so one subagent drafts it and an independent subagent challenges it before you finalize (see the `adversarial-authoring` skill). Skip it for routine changes.

## Use it

### Start a new project

```bash
git clone https://github.com/philg0ld/intent-spec-template.git my-project
cd my-project
```
Open in Claude Code and run `/opsx:propose "what you want to build"`.

### Add to an existing project

Open your project in Claude Code and ask it to follow `INSTALL_TEMPLATE.md`.

## What's inside

- `openspec/schemas/intent-spec/` — the custom schema + templates.
- `openspec/config.yaml` — activates the schema and maps each phase to a skill.
- `.agents/skills/` — reusable skills (git discipline, grill-me, Gherkin, C4, ADR, adversarial-authoring), symlinked into `.claude/skills/`.
- `.claude/commands/opsx/` + `.claude/skills/openspec-*` — generated by `openspec init --tools claude`.
- `docs/adrs/` — architecture decision records.
